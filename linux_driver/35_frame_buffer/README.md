# frame_buffer

rockchip_drm_fbdev.c为瑞芯微6.1.99内核版本实现双缓冲的参考驱动程序。

运行`make`命令后，将会有2个应用程序：

* frame_buffer_app：传统fbdev测试程序
* frame_buffer_double_app：双缓冲测试程序

运行应用程序信息：

1. frame_buffer_app

```bash
# ./frame_buffer_app

   #以下为5.5寸MIPI屏幕信息打印
   ========== 帧缓冲信息 ==========
   设备ID: rockchipdrmfb
   物理地址: 0x0, 总大小: 16588800 字节
   行宽: 4320 字节
   分辨率: 1080x1920, 位深: 32 位
   虚拟分辨率: 1080x1920
   =================================
   显存虚拟地址: 0x7f9fcee000

   开始填充黄色...
   填充蓝色...
   填充红色...
   填充绿色...
   填充白色...
   测试完成！
```

2. frame_buffer_double_app

需先根据rockchip_drm_fbdev.c修改内核支持双缓冲。

```bash
# ./frame_buffer_double_app

   #以下为5.5寸MIPI屏幕信息打印
   ========== 帧缓冲信息 ==========
   设备ID: rockchipdrmfb
   物理地址: 0x0, 总大小: 16588800 字节
   行宽: 4320 字节
   分辨率: 1080x1920, 位深: 32 位
   虚拟分辨率: 1080x3840
   =================================
   显存虚拟地址: 0x7f82cae000

   双缓冲红蓝颜色切换5次测试开始！
   显示：红色
   显示：蓝色
   显示：红色
   显示：蓝色
   显示：红色
   显示：蓝色
   显示：红色
   显示：蓝色
   显示：红色
   显示：蓝色
   测试完成！
```
